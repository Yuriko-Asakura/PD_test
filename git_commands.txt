git init
- инициализация git repo
- создание скрытой .git

git clone <адрес>
- создание рабочей копии удалённого репозитория

git status
- текущее состояние активной ветки рабочей копии репозитория

git log
- история изменений на активной ветки РКР

git remote
- информация об удалённых рабочих копиях, связанных с текущей
- origin - это обычно исходный репозиторий

git remote -v
- подробная информация


Процедура фиксации изменений

1) Добавление файлов на "сцену" (add, stage)

git add <название файла>

git add <маска>

git add .

git rm --cached
обратное от add

git commit -m first
- фиксация новой версии с сообщением после ключа -m

git diff
- просмотр изменений с последнего коммита

git diff <файл>
в конкретном файле

.gitignore - специальный файл, инструктирующий git о том, что надо игнорировать

git push - отправка неотправленных коммитов

git config -
конфигурация git

конфигурация бывает --global и --local

получение списка активных конфигураций
git --global --list

для отправки коммитов обычно нужно иметь настроенным user.name и user.email

git pull - получение неполученных коммитов

если есть изменения в одинаковых файлах, гит попробует их разрешить сам (с учётом настроек git config.pull)
если не удастся (например, буквально разные тексты на одной и той же строке), то надо разрешать конфликт вручную
-0894902uiwuir399uifuirfejui
разрешение конфликта вручную:
- удаление конфликтной разметки из конфликтного файла
- исправление логики кода в точке конфликта
- коммит результатов (конфликт разрешён локально)
- push в origin (разрешение конфликта на origin)

git branch -a
- информация о всех ветках репозитория
git branch <название>
- создание новой ветки
git checkout -b <название>
- создание новой ветки и пересадка на неё

git checkout <название>
- переход между ветками и даже отдельными коммитами без создания новых

git checkout <коммит> <файл или маска>
- выгрузка содержимого конкретных файлов из конкретного коммита


git reset - движение по истории коммитов (опасная команда, потенциально уничтожение данных, лучше не надо!)
